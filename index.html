Projects

Lazy Learned Indexes in Key-Value store based on LSM [Report]

The existing system called Bourbon based on LevelDB was using learned indices to improve the lookup latency. To make the learning resource-efficient, we proposed a new strategy called lazy learning. The lazy learned indexes to delay model learning until the first query arrives at the file. We saw 13% fewer models being learned for the workload with hot spots with lazy learning and with very minimal degradation in the latency of GET and PUT operations.

Image Compression for Computer Vision Systems [Report]

Built an image compression pipeline by retaining only the semantically important information for the target computer vision system. Depth based approach is used to identify the semantically important region. 20% savings on image file size is achieved. To measure the information loss, EAST is used to detect the information (text content) in the compressed image, and it is able to detect with an accuracy of 84%. In another approach, a separate encoding was learned for the image using FRIC network with the Inception v3 as the target computer vision system. A higher average compression rate of 46.2% with an accuracy of 52% is achieved.Technology: Python.

A study of learning knowledge graph embeddings at scale [Report]

This project is aimed to study the performance and resource consumption for learning the knowledge graph embeddings in a distributed manner for different parameters and different types of datasets.

A single-node durable key-value store

Implemented a simple key-value store with a skip list as the in-memory store and log-structured merge trees to store the data in the disk. The write-ahead logging was used to keep the data durable. Technology: C++

Performance analysis of BitWeaving Implementation in Rust

Implemented the BitWeaving (SIGMOD’13) in Rust and compared its performance with the naive scan and its original implementation C++. In this project, 128 bit SIMD instructions are used to improve and study its performance. Each element in the list is able to scan at the average cost of 2 CPU cycles per value. Technology: Rust.

Music Recommender System using Multi-Node Hadoop Cluster

We used a hybrid method for recommending songs from the dataset in the hdfs to overcome some of the disadvantages of filtering techniques when used in isolation. For content-based filtering, K-means clustering was used to cluster the songs and Pearson correlation coefficient for collaboration based filtering to find the similarity among users.

Automatic Lyric Generation based on Neuro-Linguistics Report

A research project for extending the lyric generation model (ontology graph) to generate lyrics based on author's stylometry and the backdrop context in the lyric. Word statistics for authors were built by parsing their previous works and object detection was used to build backdrop context (Caffe framework was used) and use it in the lyric.
